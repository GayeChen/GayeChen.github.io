配置改动

CommonsChunkPlugin

移除

修改为

mode

生产环境默认开启了很多代码优化（minify，splite等）
开发时开启注视和验证，并且自动加上了eval devtool
生产环境不支持watching，开发环境优化了重新打包的速度
自动设置process.env.NODE_ENV到不同环境，也就是不需要DefinePlugin来做这个了
如果你给mode设置为none，所有默认配置都去掉了

  • 方式1: 启动命令后加入参数

  • 方式2: 配置文件中加入mode属性

  • development模式下

    浏览器调试工具
    开发阶段的详细错误日志和提示
    快速和优化的增量构建机制

  • production模式下

    开启所有的优化代码
    更小的bundle大小
    去除掉只在开发阶段运行的代码
    Scope hoisting和Tree-shaking
    自动启用uglifyjs对代码进行压缩
